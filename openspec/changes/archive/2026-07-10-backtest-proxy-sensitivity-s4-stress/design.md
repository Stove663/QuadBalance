## Context

QuadBalance 已支持代理基金拼接（`BACKTEST_PROXIES`）将回测延伸到 2013-07-29，但当前只输出单一拼接路径的指标。2013–2016 年间现金、QDII、债券、黄金均依赖代理，与 2017+ 主选基金期的可信度不同。压力测试 S4 在 `stress.py` 中仅将债券象限单年收益设为 2% 的静态加权，未模拟连续 5 个日历年，与 `strategy-validation` spec 描述不符。

## Goals / Non-Goals

**Goals:**

- 量化代理跟踪误差对核心指标的影响范围（年化收益、最大回撤、验收判定）
- 按日历分段（2013–2016 代理期 vs 2017+ 主选期）报告组合表现
- 将 S4 改为对修改后价格路径的完整重模拟，债券收益连续 5 年封顶 2%
- 在 `strategy-lock.md` 和 `output/` 中输出敏感性摘要与 S4 五年路径结果

**Non-Goals:**

- 不替换现有代理映射或寻找更优代理基金
- 不改变验收门槛数值（S1–S6 条件 1–5 保持）
- 不做蒙特卡洛随机模拟（仅用确定性 ±1%/±2% 年化漂移情景）
- 不处理 P0/P1 问题（QDII 限购、分基金费率等——由 `implement-qdii-quota-simulation` 覆盖）

## Decisions

### D1: 代理敏感性扰动方式

对每条 `BACKTEST_PROXIES` 映射，在**代理段**（handoff 日之前）对日收益率施加恒定年化漂移：

- 情景：`baseline`、`+1%`、`+2%`、`-1%`、`-2%`（每条代理独立扰动，其余保持 baseline）
- 共 `1 + 4 × N` 次模拟（N = 代理条数，当前 N=5），仅对**锁定配置**运行以控制耗时

**理由**：确定性情景可复现、易写入报告；独立扰动可定位哪条代理对结论影响最大。

**备选**：Bootstrap 历史跟踪误差分布——更真实但依赖额外数据，v1 不做。

### D2: 分段报告切点

按**日历年**硬切两段：

| 分段 | 区间 | 含义 |
|------|------|------|
| `proxy_era` | 2013-01-01 – 2016-12-31 | 代理基金主导期 |
| `primary_era` | 2017-01-01 – 最新 | 主选基金主导期 |

对各段独立计算：年化收益、最大回撤、正收益年份占比。

**理由**：与用户对「2013–2016 结论可信度」的关注点对齐；不依赖 staggered handoff 日期，报告更直观。

### D3: S4 五年路径模拟

1. 取回测区间内**最后 5 个完整日历年**作为冲击窗口（如 2021–2025）
2. 复制价格矩阵，对债券象限标的（003358、003327 及 B3 混合）在窗口内逐日缩放收益，使每个日历年的年化收益 ≤ 2%
3. 对修改后价格路径**完整重跑** `simulate()`（含 DCA、再平衡）
4. 报告：5 年累计组合收益、窗口内最差单年收益、窗口年化收益

**通过标准**：5 年累计组合收益 ≥ -10%（≈ 25% 债券权重 × 2% × 5 年的保守下界，其他象限按中位数）；与现有 S4 静态情景并存，静态保留作快速参考。

**备选**：选历史债券表现最好的 5 年作为对抗窗口——v1 用最近 5 年，实现简单且代表「当下」利率环境。

### D4: 模块划分

```
src/quadbalance/
  proxy_sensitivity.py   # 扰动、分段指标、敏感性编排
  stress.py              # S4 路径化（新增 _apply_s4_bond_cap）
  sweep.py               # 锁定配置跑敏感性；S4 结果写入输出
  validation.py          # lock 文档新增 Proxy Sensitivity + S4 Path 章节
```

### D5: 输出文件

| 文件 | 内容 |
|------|------|
| `output/proxy_sensitivity.csv` | 各扰动情景下的核心指标 |
| `output/segment_metrics.csv` | proxy_era / primary_era 分段指标 |
| `strategy-lock.md` | 新增 Proxy Sensitivity Summary、S4 Five-Year Path 两节 |

## Risks / Trade-offs

| 风险 | 缓解 |
|------|------|
| 独立扰动非联合分布，可能低估多代理同时偏离 | 报告标注「独立敏感性」；后续可加联合最坏情景 |
| 恒定漂移不等于真实跟踪误差 | 用 ±1%/±2% 括号估计，分段报告补充透明度 |
| S4 窗口选择影响结果 | 报告中明确窗口年份；CLI 可选 `--s4-window` |
| 锁定配置单跑敏感性 vs 全扫描 48 配置 | v1 仅锁定配置；全扫描可选 `--full-sensitivity` |

## Migration Plan

1. 实现 `proxy_sensitivity.py` 与 S4 路径化
2. 扩展 `stress.py` / `sweep.py` / `validation.py`
3. 重跑 `uv run quadbalance`，对比 `output/` 前后
4. 无破坏性 API 变更；旧 `sweep_results.csv` 列不变，新文件追加
